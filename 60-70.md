# Documento de Requerimientos: Módulo Régimen 70 - Contenedores

**Proyecto:** Sistema de Gestión de Operaciones Marítimas - Odoo
**Módulos a Extender:**
- `ek_l10n_shipping_operations` (módulo base existente)
- `ek_l10n_shipping_operations_charging_regimes` (régimen 60/70 existente)
**Régimen Aduanero:** 70 - Depósito Aduanero (mejoras y automatización)
**Fecha:** Abril 2026
**Basado en:** Documento PDF "CAMBIOS AL 12-11-2025 CONTENEDORES"

> **NOTA IMPORTANTE:** Este proyecto **NO crea un módulo nuevo**, sino que **extiende la funcionalidad existente** del modelo `ek.operation.request` y módulos relacionados que ya están en producción.

---

## 1. RESUMEN EJECUTIVO

### 1.1 Objetivo General
Implementar un módulo completo para la gestión del **Régimen 70 (Depósito Aduanero)** que permita:
- Automatizar la extracción de datos desde documentos PDF usando Inteligencia Artificial
- Gestionar el flujo completo de contenedores desde su arribo hasta la facturación al cliente
- Controlar el inventario de repuestos/suministros para servicios de reparación de barcos externos
- Vincular el Régimen 70 con el Régimen 60 (reembarque)
- Generar plantillas de correo automáticas para notificaciones
- Facilitar la facturación a clientes externos por servicios de importación y reparación

### 1.2 Alcance
- **Usuarios objetivo:** Personal operativo de la naviera, agentes de aduanas, personal administrativo
- **Documentos procesados:** Bill of Lading (BL) y Facturas Comerciales
- **Regímenes cubiertos:** Régimen 70 (importación) con vinculación a Régimen 60 (exportación)
- **Clientes:** Externos (armadores de barcos que requieren servicios de reparación)

#### REQ-005: Creación Automática de Productos en Inventario
**Prioridad:** ALTA ⬆️
**Descripción:**
Los productos extraídos de las facturas deben crearse como **productos reales en el inventario** (`product.product`) en lugar de solo texto plano en las líneas de mercancía. Esto permite que el sistema maneje un catálogo unificado de repuestos y suministros.

**Implementación Actual:**
- El modelo `ek.boats.information` (Contenedor) tiene un campo `product_ids` (Many2many) para productos consumibles.
- El modelo `ek.operation.request` tiene un campo `regime_70_product_ids` (Many2many) para los mismos productos.
- El modelo de líneas de detalle `ek.product.packagens.goods` tiene un campo `product_id` (Many2one).

**Lógica de Búsqueda y Creación (Automática):**
Al insertar una línea vía IA o manual, el sistema debe:
1. Buscar producto existente por nombre exacto.
2. Buscar por código arancelario (HS code) en el `default_code` (formato `REG70-{HS_CODE}`).
3. Realizar fuzzy matching (similitud > 85%).
4. Si no existe, crear automáticamente como producto **CONSUMIBLE** con:
   - Categoría: "Régimen 70"
   - Tag: "Régimen 70"
   - Tipo: Consumible (no genera stock contable pero sí trazabilidad)
   - Costo: FOB unitario de la factura.

**Criterios de Aceptación:**
- ✅ Los productos se crean/vinculan automáticamente al procesar Facturas con IA.
- ✅ Los campos Many2many (`product_ids` y `regime_70_product_ids`) se pueblan automáticamente tras la extracción.
- ✅ Trazabilidad total: cada línea de mercancía apunta a un registro formal en el catálogo de productos.
- ✅ Prevención de duplicados mediante lógica de similitud en `ek.product.packagens.goods`.

---

#### REQ-005A: Implicaciones para la Inteligencia Artificial (IA)
**Descripción:**
La transición a un modelo basado en productos requiere que el motor de extracción (nsk_llm) proporcione datos más consistentes y detallados.

**Requerimientos para la IA:**
1. **Normalización de Nombres:** La IA debe intentar normalizar los nombres de los productos para facilitar el matching (ej: remover números de serie específicos si la descripción base es la misma).
2. **Extracción Precisa de HS Codes:** El código arancelario es ahora una clave de búsqueda primaria.
3. **Cálculo de FOB Unitario:** La IA debe extraer tanto el total como la cantidad para que el sistema pueda calcular el costo estándar inicial del producto.
4. **Resumen Estructurado:** El log de la IA debe informar si vinculó un producto existente o si el sistema tuvo que crear uno nuevo.


**Flujo de creación automática:**
```
1. IA extrae: "TUNAMASTER 26MM 6X195 SFC 1570/1770 RHOL COMPACTED GALVANISED A2"
   ↓
2. Sistema busca en product.product:
   - Búsqueda exacta: No encontrado
   - Fuzzy match (>85%): ¿"TUNAMASTER 26MM" existe? SÍ → Confianza 90%
   ↓
3. Sistema muestra sugerencia:
   "Producto similar encontrado: TUNAMASTER 26MM RHOL COMPACTED"
   [✓ Usar este producto] [Crear nuevo producto] [Buscar otro]
   ↓
4. Usuario selecciona opción
   ↓
5. Línea se crea con product_id vinculado
```

**Beneficios:**
- ✅ Sin duplicados de productos con descripciones ligeramente diferentes
- ✅ Trazabilidad: Ver histórico de importaciones por producto
- ✅ Análisis: Qué productos se importan más frecuentemente
- ✅ Costos: Histórico de precios FOB por producto
- ✅ Proveedores: Qué proveedores suministran cada producto
- ✅ Facturación: Usar lista de precios estándar al facturar a clientes

**Flujo del Wizard:**

   - Extraer datos con IA si no se ha hecho

2. **Comparación Automática:**
   - Crear línea de wizard por cada producto de factura
   - Buscar producto correspondiente en Nota de Pedido

   - **Validar Seleccionadas**: Solo las líneas marcadas con `validate_anyway`

5. **Resultado:**
   - Actualizar `validation_status` en `ek.product.packagens.goods`
   - Actualizar `po_line_reference` con número de ítem
   - Guardar `validation_notes` con diferencias

#### 1B. REQ-002: Identificación automática del tipo de documento

**Problema actual:**
Existen dos acciones separadas (`action_extract_bl_with_ai`, `action_extract_invoices_with_ai`) que el usuario debe invocar manualmente. Si sube un PDF genérico sin saber su tipo, no hay flujo de auto-detección.

**Qué agregar:**
Un método `action_auto_detect_and_extract` que:

1. Llama al LLM con un prompt de clasificación (sin tool calling, respuesta corta):

```python
def _detect_document_type(self, attachment):
    """
    Detecta si un PDF es BL, Factura Comercial o Nota de Pedido.
    Devuelve: 'bl' | 'invoice' | 'purchase_order' | 'unknown'
    """
    llm = self.env['nsk.llm.provider']
    messages = [
        {
            "role": "system",
            "content": (
                "Eres un experto en documentos aduaneros. "
                "Responde ÚNICAMENTE con una de estas palabras: "
                "bl, invoice, purchase_order, unknown"
            )
        },
        {
            "role": "user",
            "content": (
                "¿Qué tipo de documento es este PDF? "
                "Opciones: bl (Bill of Lading), invoice (Factura Comercial), "
                "purchase_order (Nota de Pedido), unknown."
            )
        }
    ]
    response = llm.generate_completion(
        messages=messages,
        attachments=attachment
    )
    raw = response.choices[0].message.content.strip().lower()
    if raw in ('bl', 'invoice', 'purchase_order'):
        return raw
    return 'unknown'
```

2. Dependiendo del resultado, enruta a `action_extract_bl_with_ai`, `action_extract_invoices_with_ai` o lanza `UserError` pidiendo clasificación manual.

3. Si el PDF ya tiene tipo definido (campo `document_type` en el registro), saltar la detección.

#### 2A. REQ-005: Robustez de `_find_or_create_product` con HS codes

**Problema actual:**

```python
def _check_duplicate_lines(self):
    """
    Detecta líneas de mercancía potencialmente duplicadas.
    Criterio: misma description + misma quantity + mismo total_fob.
    Lanza UserError con lista de duplicados.
    REQ-006
    """
    lines = self.ek_produc_packages_goods_ids
    seen = {}
    duplicates = []

    for line in lines:
        key = (
            (line.name or '').strip().upper(),
            round(line.quantity or 0, 3),
            round(line.total_fob or 0, 2),
        )
        if key in seen:
            duplicates.append((seen[key], line))
        else:
            seen[key] = line

    if duplicates:
        detail = '\n'.join(
            f'  • Línea {a.id} y Línea {b.id}: "{a.name}" — '
            f'{a.quantity} uds — FOB ${a.total_fob:.2f}'
            for a, b in duplicates
        )
        raise UserError(_(
            'Se detectaron %s pares de líneas posiblemente duplicadas:\n\n%s\n\n'
            'Consolide o elimine las líneas duplicadas antes de continuar.'
        ) % (len(duplicates), detail))
```

**Problema actual:**

**A) Implementar comparación determinista en `_fallback_manual_comparison`:**

```python
def _fallback_manual_comparison(self, invoice_lines, po_items):
    """Comparación híbrida: texto + numérica determinista."""
    WEIGHT_TOL_KG = 0.5    # tolerancia peso: ±0.5 kg
    FOB_TOL_PCT   = 0.05   # tolerancia FOB: ±5%
    TEXT_MATCH    = 0.70   # umbral mínimo de similitud de descripción

    validation_lines = []
    for inv_line in invoice_lines:
        best_ratio, best_match_idx, best_po = 0, -1, None

        for idx, po_item in enumerate(po_items):
            ratio = SequenceMatcher(
                None,
                (inv_line.name or '').lower(),
                po_item.get('description', '').lower()
            ).ratio()
            if ratio > best_ratio:
                best_ratio, best_match_idx, best_po = ratio, idx, po_item

        diffs = []
        status = 'missing'

        if best_ratio >= TEXT_MATCH and best_po:
            # Comparación numérica de peso
            inv_w = inv_line.gross_weight or 0
            po_w  = best_po.get('weight_kg', 0) or 0
            if po_w and abs(inv_w - po_w) > WEIGHT_TOL_KG:
                diffs.append(
                    f'Peso: Factura={inv_w:.2f} kg vs PO={po_w:.2f} kg '
                    f'(dif: {abs(inv_w - po_w):.2f} kg)'
                )

            # Comparación numérica de FOB
            inv_fob = inv_line.total_fob or 0
            po_fob  = best_po.get('fob_value', 0) or 0
            if po_fob and abs(inv_fob - po_fob) / po_fob > FOB_TOL_PCT:
                diffs.append(
                    f'FOB: Factura=${inv_fob:.2f} vs PO=${po_fob:.2f} '
                    f'(dif: {abs(inv_fob - po_fob) / po_fob * 100:.1f}%)'
                )

            if not diffs:
                status = 'match'
            elif len(diffs) == 1 and 'Peso' in diffs[0]:
                # Solo diferencia de peso pequeña → menor
                status = 'minor'
            else:
                status = 'major'

        val_line = {
            'goods_line_id': inv_line.id,
            'invoice_description': inv_line.name or '',
            'invoice_quantity': inv_line.quantity or 0,
            'invoice_weight': inv_line.gross_weight or 0,
            'invoice_fob': inv_line.total_fob or 0,
            'validation_status': status,
            'differences': '\n'.join(diffs) if diffs else (
                f'Similitud: {best_ratio*100:.0f}%' if best_po else 'No encontrado en Nota de Pedido'
            ),
        }
        if best_po:
            val_line.update({
                'po_item_number': best_po.get('item_number', ''),
                'po_description': best_po.get('description', ''),
                'po_quantity': best_po.get('quantity', 0),
                'po_weight': po_w,
                'po_fob': po_fob,
            })
        validation_lines.append((0, 0, val_line))

    self.validation_line_ids = validation_lines
```
